# Source Map

This source map explains where each major fact in this docs repo comes from. It is intentionally short so humans and AI agents can decide whether a claim needs verification in official docs before production use.

## Fact Groups

Project category: Stirling PDF is presented publicly as an open-source PDF platform. Use upstream README and official docs for the current category wording.

Core capabilities: edit, sign, redact, convert, OCR, compress, merge, split, automation, and related PDF tools. Use official functionality docs for the current full list.

Tool count: official docs describe 60+ tools. Treat this as current observed docs wording, not a permanent invariant.

Deployment modes: public docs describe desktop applications, browser UI, Docker deployment, manual server setup, Kubernetes, self-hosted server, and private API paths.

Manual server dependency: docs mention Java 21+ for manual server setup, with dependencies such as LibreOffice and Tesseract for OCR-related work.

API: official docs describe local Swagger UI, API authentication, global API key configuration, REST API use from automation platforms, limitations for frontend-only operations, and pipeline automation.

License: upstream license text describes an open-core structure with MIT for most repository content outside listed restricted directories.

Hosted planner relationship: Stirling PDF Space is independent and unofficial. It helps plan workflows around the project at https://stirling-pdf.space/ and must not be presented as upstream.
